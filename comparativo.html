Conclusión

Gracías al cuadro comparativo podemos notar que el más antiguo entre Git, Mercurial y Subversión es este ultimo que les tiene una ventaja de 7 y 5 años respectivamente, podemos notar que git y Mercurial estan en situaciones similiares referentes a la peridiocidad de sus actualizacion a diferencia de Subversion que tiene mucho mas tiempo sin actualizarse, algo en que todos estos controladores de versiones tienen en comun es que todos fueron codificados en lenguaje C. La finalidad de crear un cuadro comparativo sobre tres software de control de versiones existentes, como Git, Subversion y Mercurial, sería proporcionar una visión objetiva de cada uno de ellos. Siendo de gran ayuda para decidir cual de esos sistemas de control de versiones se ajusta mejor a mis necesidades, por otr lado tambien ayuda a conocer otras alternativas aparte de git
